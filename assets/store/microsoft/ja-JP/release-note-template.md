# Microsoft Store リリースノート — スタイルガイド

このファイルは、AI がリリースノートを生成する際の指示書です。
人間が手動で書く場合もこのガイドラインに従ってください。

## 出力形式

- 日本語のみ（英語・記号は最小限）
- 箇条書き形式。各項目は `・` で始める
- ヘッダー・HTML タグ・Markdown 記法は使用しない
- 全体で 500 文字以内（Microsoft Store の上限）
- 項目数は 3〜6 個が理想

## 文体

- 敬体（「〜しました」「〜できます」形式）
- ユーザー目線の表現（開発者向け内部変更は除外）
- 変更を過去形で説明：「〜を改善しました」「〜を追加しました」「〜を修正しました」

## 含めるべき変更

- ユーザーが直接体験できる新機能
- UI の改善や操作性の向上
- ユーザーに影響するバグ修正

## 除外すべき変更

- CI/CD・テスト・ビルドシステムの変更
- 内部リファクタリング（ユーザー影響なし）
- ドキュメント・README の変更
- 依存パッケージのバージョン更新（ユーザー影響なし）

## フォールバック

ユーザーに影響する変更が見当たらない場合は、以下をそのまま出力してください：

動作の安定性を向上しました

## 出力例（良い例）

・縦書きモードの表示ズレを修正しました
・ルビ入力のキーボードショートカットを追加しました
・読み上げ速度の調整が即座に反映されるようになりました
