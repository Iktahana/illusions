# Microsoft Store Listing — Source of Truth

`store/microsoft/ja-JP/` contains all Japanese listing content for the Microsoft Store.
Edit these files, then run the **Sync Microsoft Store Listing** workflow to publish.

> **Important**: This workflow can only be run from the `main` branch.
> Merge your changes to `main` before triggering.

## Files

| File                             | Store field            | Notes                                                                                                 |
| -------------------------------- | ---------------------- | ----------------------------------------------------------------------------------------------------- |
| `ja-JP/description.md`           | Description (long)     | Main product description. Plain text with emoji. Max 10,000 chars.                                    |
| `ja-JP/short-description.md`     | Short description      | One-line summary shown in search results. Max 1,000 chars (270 chars 未満推奨).                       |
| `ja-JP/features.md`              | Features list          | Bullet list (`- item`). Each item becomes a feature entry. Max 200 chars/item.                        |
| `ja-JP/release-note-template.md` | Release notes template | AI system prompt + style guide. Release notes are auto-generated by the workflow and never committed. |

## Workflow

1. Edit the files above, commit to your branch, and merge to `main`
2. Go to **Actions > Sync Microsoft Store Listing**
3. Select branch **main** and click **Run workflow**
4. Choose mode:
   - **listing-only** — syncs text/metadata only. No package upload. Use this for copy edits.
   - **full-submission** — syncs text AND uploads `.appx` packages from a GitHub Release. Provide the `release_tag` (e.g. `v0.2.0`).
5. Optionally enable **dry_run** to preview without making changes

## When to use which mode

| Scenario                    | Mode              | release_tag |
| --------------------------- | ----------------- | ----------- |
| Fixed a typo in description | `listing-only`    | —           |
| Updated feature list        | `listing-only`    | —           |
| New app version released    | `full-submission` | `v0.2.0`    |

## Important: pending submissions

If Partner Center already has a **pending submission** (draft, in-progress package upload, etc.):

- **listing-only** will **fail immediately** rather than risk committing unrelated draft changes. You must complete or delete the pending submission in Partner Center first, then re-run.
- **full-submission** will reuse the pending submission by default, or delete it first if `FORCE_DELETE_PENDING=true` is set.
