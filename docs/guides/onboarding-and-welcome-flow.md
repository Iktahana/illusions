# オンボーディングとウェルカムフロー

illusions を初めて起動した際や、何もファイルを開いていないときのユーザー体験（UX）を定義する「ウェルカム画面」と「オンボーディング」のフローについて説明します。

## ウェルカム画面 (`WelcomeScreen`)

アプリケーションを起動し、何もファイルやプロジェクトが読み込まれていないときに表示される初期画面です。

### 主な役割

- **新規作成**: 「新規プロジェクト」ウィザードを起動します。
- **既存を開く**: OS のダイアログからプロジェクトフォルダ、または単一のファイル（スタンドアロン）を開きます。
- **履歴からの復元**: 「最近のプロジェクト」一覧から、過去に開いたプロジェクトに素早くアクセスできます。
- **アカウント管理**: ログイン、マイページへのリンク、設定へのアクセスを提供します。

## 初回起動時のフロー

1. **アプリ起動**: 起動時、`StorageService` から「前回のセッション」で開いていたプロジェクトの有無を確認します。
2. **自動復元 (`useAutoRestore`)**:
   - 前回のプロジェクトがある場合: 自動的に読み込みを試みます（Web版では権限の再確認ダイアログが表示されることがあります）。
   - 前回のプロジェクトがない、または復元に失敗した場合: ウェルカム画面を表示します。
3. **新規プロジェクト作成**:
   - ユーザーが「新規プロジェクト」を選択すると、`CreateProjectWizard` が開きます。
   - プロジェクト名と保存場所を決定すると、`.illusions/` フォルダを含む標準的な構成が作成され、エディタが開きます。

## 新規ウィンドウと `?welcome` パラメータ

illusions では、複数のウィンドウをサポートしています（Electron 環境）。

- **新規ウィンドウの生成**: 通常、新しいウィンドウを開く際は前回の状態が自動復元されます。
- **意図的なウェルカム表示**: `?welcome` クエリパラメータを URL に付加して開くことで、自動復元をスキップして強制的にウェルカム画面を表示させることができます。

## 権限の許可と誘導

Web 版では、ブラウザのセキュリティ制約（File System Access API）により、ユーザーがフォルダへのアクセスを許可する「明示的なアクション」が必要です。

- **`PermissionPrompt`**: プロジェクトを自動復元しようとした際、権限が失効している場合に表示されます。
- **誘導メッセージ**: 非サポートブラウザでの閲覧時には、デスクトップ版のダウンロードを促すメッセージが表示されます。
