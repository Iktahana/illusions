# 省電力対応ガイド

Issue #1402 対応として、実バッテリー計測ではなく「CPU 使用率・タイマー wake-up・バックグラウンド処理頻度」を減らす観点で判断する。

## 対象にすべき処理

- 定期 timer / interval / polling
- 形態素解析や全文走査のような高コスト解析
- 表示補助や装飾のための処理
- 非表示ウィンドウや非アクティブ状態でも継続する監視

## 止めてよい処理

- POS ハイライトのような装飾的処理
- 可読性補助スコアのような非必須解析
- 手動更新で代替できる目次・統計の定期再計算

## 止めすぎてはいけない処理

- autosave / recovery
- ユーザー操作に直結する保存・表示・入力
- セキュリティや整合性のために必要な処理

これらは停止ではなく「低頻度化」を優先する。

## 判断基準

新機能を追加するときは、次を確認する。

1. その処理はユーザーが今見ていない間にも走るか
2. その処理は結果が少し遅れても安全か
3. その処理は UI の装飾や補助分析に留まるか
4. 停止ではなく低頻度化で安全性を保てるか

`powerSaveMode` またはウィンドウのバックグラウンド化で安全に止められる処理は、省電力対象に含める。
