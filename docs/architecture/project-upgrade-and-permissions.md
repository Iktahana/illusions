# プロジェクトのアップグレードと権限

illusions では、単一のファイルを開く「スタンドアロンモード」から、専用のディレクトリ構造を持つ「プロジェクトモード」への移行（アップグレード）機能を提供しています。また、ブラウザ版では File System Access API に基づく権限管理を適切に行う必要があります。

## 設計の目的

- **柔軟な導入**: ユーザーが単一ファイルの編集から始め、必要に応じてプロジェクトに昇格できるようにする。
- **セキュリティの確保**: ブラウザが要求するフォルダへのアクセス権限を管理する。
- **セッションの継続**: プロジェクトを再オープンする際、権限の再確認をスムーズに行えるようにする。

## プロジェクトのアップグレードフロー

ユーザーが「プロジェクトにアップグレード」を選択すると、以下のプロセスが実行されます。

1. **名前の提案**: 現在開いているスタンドアロンファイルのファイル名から、プロジェクト名を自動抽出します。
2. **ディレクトリ作成**: `ProjectService.createProject()` を呼び出し、OS のディレクトリピッカーを表示します。
3. **プロジェクト構造の初期化**:
   - `.illusions/` ディレクトリを作成します。
   - `project.json` (基本設定) と `workspace.json` (タブ配置などの状態) を作成します。
   - `history/index.json` (履歴インデックス) を空の状態で作成します。
4. **ファイル移行**:
   - 元のファイルの内容を、プロジェクトルートに `${projectName}.${extension}` として保存します。
5. **設定の継承**: スタンドアロンモードで設定していたエディタ設定を、新しいプロジェクトのメタデータに引き継ぎます。
6. **VFS 切り替え**: アプリケーションの Virtual File System (VFS) のルートを新しく作成されたディレクトリに設定し、プロジェクトモードとして再初期化します。

## 権限管理 (Web 版)

Web 版では File System Access API を使用するため、セキュリティ上の制約により、セッションごとに権限（PermissionHandle）の有効性を確認する必要があります。

### 権限の再確認 (`PermissionPrompt`)

ブラウザはセキュリティのため、ページの再読み込み時にフォルダへのアクセス権限をリセットします。

- **検知**: プロジェクトを再読み込みする際、`queryPermission` が `prompt` 状態であることを検知します。
- **UI 表示**: ユーザーに対して「アクセス許可が必要です」というダイアログを表示します。
- **実行**: ユーザーが「許可する」をクリックした際、`requestPermission` を呼び出してネイティブの権限許可ダイアログをトリガーします。

### Electron 版での違い

Electron 版ではファイルシステムへの直接アクセス権があるため、これらの権限確認ダイアログは表示されません。

## 最近のプロジェクトの復元

最近開いたプロジェクト（Recent Projects）から再オープンする場合のフローは以下の通りです。

1. **エントリ選択**: ホーム画面の「最近のプロジェクト」からエントリを選択します。
2. **Handle の取得**: `StorageService` に保存されていたディレクトリハンドル（IndexedDB）を復元します。
3. **権限確認**: 保存されていたハンドルに対して `requestWritePermission` を呼び出し、必要に応じてユーザーに許可を求めます。
4. **プロジェクトのロード**: 権限が取得でき次第、VFS をマウントしてプロジェクトの内容をエディタに展開します。
